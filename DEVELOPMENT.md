## 一、产品定位

**一句话：** 帮中国 NBA 球迷在偶像退役的过渡期，用直觉找到「下一个十年该看的球员」。

| 维度 | 结论 |
|------|------|
| 用户 | 泛球迷（入门探索）+ 深度球迷（验证判断）双覆盖 |
| 情感钩子 | 中代球迷面临詹姆斯/库里/杜兰特退役，找下一个篮球信仰 |
| 产品形态 | 以球员为主体的半社交半信息平台 |
| 壁垒 | 短期先发卡位 → 长期 Prompt + 数据库优势 |

## 三、当前架构

### 匹配引擎 (V3)
```
用户选球星 → 13D 向量平均 → DNA
     │
     ├── 50% 13D 余弦相似度
     ├── 25% 位置匹配
     ├── 25% 身体匹配（身高/臂展/体重）
     └── +4 分即战力/潜力股偏好加成
     │
     ▼
综合排名 + 潜力股保底(≥1)
```

### 用户展示 (6 组融合)
终结(蓝) / 投射(绿) / 组织(黄) / 防守(红) / 身体(白·独立横条) / 篮板(紫)

### 饼图 (5 组 donut，身体独立)
- 饼图用 `Math.pow(value, 1.8)` 非线性放大
- 前三名扇区 ×1.2、后二 ×0.75 视觉拉伸
- paddingAngle=0 整环无缝，深色细线分割
- ⚠️ 此格式已定版，别改

### 部署
```
用户 → Cloudflare Pages (前端) → Railway (后端) → DeepSeek API
       next-star-5s9.pages.dev    :8080             api.deepseek.com
```

## 四、数据流

| 数据 | 来源 | 格式 |
|------|------|------|
| 球星模板 36 人 | `src/data/star-players-13d.json` | 13D 属性 + body + skills/style |
| 新秀库 36 人 | `src/data/2026-draft-database.json` | 13D 属性 + tags + combine + profile_text_cn |
| AI 搜索 | DeepSeek API (server/index.js) | 13D 向量 via VECTOR_PROMPT |
