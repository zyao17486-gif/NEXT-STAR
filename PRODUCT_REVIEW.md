# Next Star 核心系统审查

> 2026-06-21 · 不讨论技术实现，只审查产品设计

---

## 1. 用户画像 — 评分 6/10

### 当前设计

```
Step 1: 选位置 (PG/SG/SF/PF/C)
Step 2: 选一个你欣赏的球星 (5人/位置 × 5位置 = 25人)
Step 3: 天赋型 还是 勤奋型？
```

### 合理之处

- **"选球星"替代"填属性"是聪明的设计**。普通球迷不会说"我要投篮 85 分、防守 70 分的侧翼"，但他们知道"我喜欢伦纳德那种球员"
- **3 步不冗长**，完成率应该不低

### 问题

**① 单选球星过于粗暴**

用户选"库里" → 系统把库里的 5 维属性当成用户偏好。但用户欣赏库里的到底是什么？是三分投射？是无球跑动？是领袖气质？还是单纯喜欢勇士队？

> **风险**：一个选择承载了太多未经证实的假设。

**② 天赋/勤奋二元对立是伪命题**

没有球迷会认为自己是"纯天赋型"。这个维度的权重调整只有 ±10%，实际影响微弱——但它占了一个 Step，让用户觉得"我要做一个奇怪的选择题"。

**③ 缺少路人友好路径**

当前入职假设用户认识 25 个球星中的至少 1 个。一个刚接触 NBA 的用户走到 Step 2 会卡住。虽然有 Skip 按钮，但它藏在角落里，用户不一定会注意到。

**④ 缺少"主队"维度**

大量球迷的选秀兴趣来自"我的主队需要什么位置"。如果用户是火箭球迷，他们最想知道"2026 年选秀中谁适合火箭"——当前系统无法理解这个意图。

### 改进建议

| 优先级 | 改动 |
|--------|------|
| P0 | Step 2 增加"随便看看 / 不太了解"选项（路人友好） |
| P1 | Step 2 支持多选 2-3 个球星，加权混合（"50% 库里 + 50% 欧文"） |
| P1 | 天赋/勤奋替换为"即战力 🆚 潜力股"（球迷更熟悉的维度） |
| P2 | 可选：输入你的主队（球队需求匹配） |
| P2 | 可选：你的篮球知识水平（新手 / 中级 / 深度）→ 影响结果解释的复杂度 |

## 2. DNA 系统 — 评分 7/10

### 当前设计

```
5 维属性向量：[终结, 投篮, 组织, 防守, 运动]
        ↓
球星选择 → 逐元素平均 → 天赋/勤奋加权(±10%) → 归一化
```

### 合理之处

- **5 维覆盖了篮球核心技能**，不是随意选的
- 每一维都有清晰的球探定义（见 prompt 中的英文描述）
- 余弦相似度简单透明，不需要训练数据

### 问题

**① 缺少球商维度（BBIQ）**

球探报告中最常出现的词之一就是"篮球智商 / 比赛感觉 / 决策能力"。当前的 5 维里这一项完全缺失。playmaking ≠ BBIQ——一个球员传球好不一定决策好。

**② 篮板维度不一致**

球员详情页的能力饼图包含 `rebounding`（篮板），但 DNA 系统中没有这一维。用户 DNA 是 5 维，球员也是 5 维，但展示给用户的是 6 维饼图——逻辑断裂。

**③ 维度等权不合理**

余弦相似度对所有维度一视同仁。但现实中：
- 控卫的组织能力权重大于中锋的组织能力
- 中锋的终结/防守权重大于后卫
- 不同球星偏好暗示了不同的维度重要性

**④ 维度之间存在冗余**

`finishing` 和 `athleticism` 天然相关（运动能力强的球员通常终结也更好）。在等权余弦相似度下，运动天赋强的球员会在两个相关维度上同时得分，造成匹配偏差。

### 改进建议

| 优先级 | 改动 |
|--------|------|
| P0 | 将 rebounding 加入 DNA 系统（6 维），或从饼图中移除（统一为 5 维） |
| P1 | 按位置动态调整维度权重（例如 PG 的 playmaking 权重 ×1.3） |
| P1 | 增加 BBIQ 维度（可从球探标签中提取 proxy 值："high IQ""cerebral"→高 BBIQ） |
| P2 | 对相关维度（finishing×athleticism）应用降权，减少冗余影响 |

## 3. 应该增加的字段 — 球员 & 用户

### 球员字段

| 字段 | 为什么需要 | 数据来源 |
|------|-----------|----------|
| **BBIQ / 比赛感觉** (0-100) | 球探最看重的品质之一，当前系统缺失 | 标签提取（"high IQ""cerebral"）或 LLM 评估 |
| **Motor / 拼劲** (0-100) | "防守积极性起伏""动力存疑"在球探报告中反复出现 | 同上 |
| **身体静态分数** (0-100) | 身高+臂展+体重的综合评分。一张饼图看不出来这人是 6'3" 还是 6'10" | 体测数据加权合成 |
| **伤病史标记** (enum) | 选秀顺位的核心变量。健康的新秀和伤过的新秀是完全不同的投资 | 球探报告提取（"injury risk""injury recovery"） |
| **年龄** (number) | 大一（18-19岁）vs 大四（22-23岁）选秀前景差异巨大 | 已有 `birthday` 但未参与计算 |
| **生产力分数** (0-100) | 大学数据综合评分（PER / BPM / 场均得分），数据驱动的客观维度 | 赛季统计数据合成 |

### 用户字段

| 字段 | 为什么需要 | 如何使用 |
|------|-----------|----------|
| **主队** | "我的主队需要什么位置"——这是球迷最自然的选秀问题 | 匹配时引入球队需求权重 |
| **篮球知识水平** | 新手需要更多解释，深度球迷需要更多数据 | 决定详情页"数据"标签页默认展示 vs "概览" |
| **偏好风格多样性** | 用户可能喜欢多种类型（防守型+3D 型） | 支持多维度偏好画像 |

## 4. 推荐逻辑 — 评分 5/10

### 当前设计

```
用户 DNA (5D) × 球员属性 (5D) → 余弦相似度
    ├── 属性相似度 80%
    └── Embedding 相似度 20%
        ↓
同位置优先 → Top 3
```

### 问题

**① 位置优先太刚性**

当前逻辑：同位置球员排完才考虑其他位置。

> 用户选了 PG → 永远看不到 SG 的推荐。但如果 2026 届最好的进攻发动机恰好是 SG，用户错过了最适合他的球员。

这是个严重问题。**位置优先应该是加分项，不是过滤器。**

**② Top 3 可能高度同质化**

没有任何多样性约束。用户可能得到 3 个几乎一模一样的推荐——"3 个防守型侧翼"。正确的推荐应该展示不同风格：1 个最匹配 + 1 个同风格备选 + 1 个不同风格的意外发现。

**③ 80/20 权重没有校准依据**

为什么是 80/20？如果 embedding 是从属性向量派生的，那 20% 的 embedding 权重就是变相重复计数。需要验证 embedding 和 attributes 是否正交。

**④ 缺少"意外惊喜"机制**

最好的推荐不只是"找到你已知想要的"，还包括"发现你不知道自己会喜欢的"。当前纯数学逻辑不会产生 serendipity。

**⑤ 只返回 3 个结果**

用户想浏览更多怎么办？没有"查看更多推荐""换成其他风格""我不喜欢这些，换一批"的出口。

**⑥ 搜索和推荐是两套逻辑**

球探搜索用的是 LLM → 向量 → 相似度（3 阶段流水线）。入职推荐用的是纯数学余弦相似度。两套系统没有共享用户 DNA 权重。球探搜索可以混合用户 DNA（7:3），但入职推荐没有用 LLM 增强解释——两套逻辑各做各的，体验割裂。

### 改进建议

| 优先级 | 改动 |
|--------|------|
| **P0** | **位置加权替代位置过滤**：同位置 +15% 分数加成，但不排除其他位置 |
| **P0** | **Top 3 多样性约束**：确保 3 个推荐覆盖至少 2 个不同风格/位置 |
| **P1** | **Top 3 中包含 1 个"意外发现"槽位**：分数中上但风格不同的球员 |
| **P1** | **基于所选球星动态调节维度权重**：选库里 → shooting 权重 ×1.3；选字母哥 → athleticism+finishing 权重 ×1.3 |
| **P1** | **返回至少 5 个结果**：Top 3 精确匹配 + 2 个扩展推荐 |
| **P2** | **验证 80/20 权重**：用 A/B 测试对比不同权重下的用户满意度 |
| **P2** | **统一搜索和推荐的用户 DNA 混合逻辑**：入职推荐也应该用 LLM 增强解释 |

---

## 总结

| 系统 | 当前评分 | 最主要问题 | 最先改什么 |
|------|----------|-----------|-----------|
| 用户画像 | 6/10 | 单选球星太粗暴；天赋/勤奋无意义 | 加"随便看看"路径；天赋/勤奋→即战力/潜力股 |
| DNA 系统 | 7/10 | 缺球商维度；篮板维度不一致 | 统一 5 维或加回篮板到 DNA |
| 数据字段 | 5/10 | 缺年龄、伤病史、球商、身体静态分 | 身体静态分数（最易合成）；年龄（已有数据未用） |
| 推荐逻辑 | 5/10 | 位置过滤太刚性；无多样性；无 serendipity | 位置加权替代过滤；Top 3 多样性约束 |
