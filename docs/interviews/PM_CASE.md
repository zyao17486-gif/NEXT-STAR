# Next Star — 产品案例

> 面试 3 分钟叙述框架：问题 → 假设 → 方案 → 迭代 → 反思

## 1. 我解决什么问题

**核心洞察：中国 NBA 球迷在选秀季存在信息鸿沟。**

- 每年 60 名新秀，主流球探报告全是英文
- 中文球迷的信息源是虎扑翻译帖和抖音碎片视频
- 传统 mock draft 网站（Tankathon、ESPN）按排名罗列，不提供个性化推荐
- 球迷最自然的提问方式——"有没有像伦纳德那样的新秀"——没有任何产品能回答

**一句话问题定义：** 中国 NBA 球迷需要一个无需英文能力、不依赖排名、用直觉方式发现可能喜欢的新秀的工具。

## 2. 我的核心假设

> **"选择你欣赏的球星"比"填写属性筛选器"的推荐完成率高 3 倍以上。**

推理：
- 球迷不熟悉"终结能力 85 分"这种抽象维度
- 但每个球迷都知道"我喜欢库里/字母哥/伦纳德"
- 球星选择隐含了丰富的多维偏好（风格、态度、位置、球风）
- 一个选择承载了传统推荐系统需要 10 个问题才能采集的信息

## 3. 我的方案

```
用户选"我喜欢库里" → 系统提取库里 6D 属性向量
→ 余弦相似度匹配 36 名新秀 → Top 4 推荐
→ LLM 为每个推荐生成中文解释
```

**为什么是余弦相似度而不是 ML 模型？**
- 36 人数据库太小，训练任何模型都会过拟合
- 余弦相似度完全透明——每个推荐可以追溯到"因为你的库里偏好让你的投篮维度权重更高"
- 用户可以理解为什么被推荐某球员，信任感远超黑箱推荐

**为什么用 LLM？**
- 匹配分数只告诉用户"有多像"，LLM 告诉用户"为什么像"
- 中文球探报告翻译——这是传统方案做不到的独特价值

## 4. 关键迭代

| 版本 | 发现的问题 | 改动 | 预期影响 |
|------|-----------|------|---------|
| V1.0 | 位置过滤导致 PG 用户永远看不到 SG | 位置过滤 → 位置加权（+15%） | 推荐多样性↑ |
| V1.1 | Top 3 可能全是同类型球员 | 多样性约束：至少覆盖 2 种风格 | 用户探索深度↑ |
| V1.2 | 匹配分数显示偏高（100% 不真实） | 展示原始相似度，加成仅用于排序 | 信任度↑ |
| V1.2 | 天赋/勤奋二元不合直觉 | 改为即战力 vs 潜力股 | 入职完成率预期↑ |

## 5. 还没解决但应该解决的问题

- **冷启动**：不认识任何球星的路人用户怎么办？
- **验证闭环**：推荐准确吗？用户满意吗？——缺反馈机制
- **评委盲区**：算法推荐 vs 专家榜单有多大差异？——缺对比基准

## 6. 如果重来一次我会怎么做

不是从功能列表出发，而是从**用户任务**出发：

1. "帮我找到我会喜欢的新秀" → 入职推荐
2. "这个球员到底什么水平" → 球员详情
3. "有没有我没想到但值得关注的" → 多样性推荐
4. "我这周错过了什么" → 趋势变化

每个任务一个入口，而不是把功能堆在一个侧边栏里。
