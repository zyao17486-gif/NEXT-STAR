# 球员数据编写规范

适用于往 `2026-draft-database.json` 新增球员时，编写以下 5 个字段。

## 核心原则

- **叙述客观**：禁止「精英级」「统治比赛」「令人震惊」「天花板最高」「无可匹敌」「名副其实」等情感词汇
- **引用数据**：Synergy 百分位、具体命中率、场均统计、联合试训排名
- **模板独立**：综述中不出现模板对比（模板放在 ceilingTemplate / floorTemplate 中）

---

## 字段模板

### 1. `profile_text_cn` — 球员综述（3-4 句）

结构：身体/位置 → 打法/数据 → 亮点 → 短板/风险

```
{姓名}是一名身高{X尺X寸}、{体重/臂展}的{位置}，{大学/联赛}期间{核心打法概括}。
{关键数据支撑（场均、命中率、Synergy百分位）}。
{最突出的一项能力及数据佐证}。
{主要短板或选秀风险，客观陈述不夸张}。
```

示例：
> AJ·迪班萨是一名身高6尺9寸、臂展7尺0.5寸的侧翼得分手，在杨百翰大学大一赛季领跑NCAA得分榜。他的进攻以自主创造中距离跳投和持续篮下施压为核心——篮下出手仅23%来自助攻，造罚球次数领跑全美。防守端拥有身高臂展和横移基础，但无球防守的专注度和积极性存在明显波动。三分投射弧线偏平（33-34%命中率）是当前进攻完整性的主要制约。

### 2. `ceilingTemplate` — 上限模板

```json
{ "name": "真实NBA球员名", "desc": "一句条件+结果。格式：若{条件}，{结果}" }
```

示例：
```json
{ "name": "Jayson Tatum", "desc": "三分投射弧线改善至联盟平均水平以上（36%+），无球防守习惯和专注度提升至可依赖水准，发展为兼具持球核心产量与季后赛防守兼容性的全明星侧翼。" }
```

### 3. `floorTemplate` — 下限模板

```json
{ "name": "真实NBA球员名", "desc": "一句风险+结果。格式：若{风险}，{结果}" }
```

示例：
```json
{ "name": "Andrew Wiggins（勇士版本）", "desc": "三分投射始终未突破34%，无球防守专注度未能持续改善，进攻效率依赖高出手量支撑，最终定位为需要特定阵容搭配的中产得分手而非球队第一选择。" }
```

### 4. `strengths_cn` — 优势（3-4 条）

每条一条完整的客观叙述句，引用具体数据。

示例：
```json
[
  "三威胁得分覆盖面无明显盲区——中距离脚步稳定，篮下终结通过身体控制和造犯规能力维持高效率（罚球次数领跑NCAA）。",
  "挡拆（87百分位）和背身（94百分位）两个核心半场进攻场景的效率均位居NCAA前列。",
  "身体硬件突出：6尺9寸身高搭配7尺0.5寸臂展和42英寸最大弹跳，兼具前锋骨架和后卫移动流畅度。"
]
```

### 5. `weaknesses_cn` — 待观察（3-4 条）

每条指出一个具体的、可验证的短板，忌用「稳定性待验证」「防守端需持续提升」等空泛表述。

示例：
```json
[
  "三分投射弧线偏平导致命中率仅33-34%，对手在挡拆中倾向于绕掩护放投。",
  "无球防守投入度波动明显——扑防外线迟缓、对手转移球时站位松散。",
  "持球突破面对针对性收缩防守时失误率偏高，决策速度和控球稳定性尚不足以稳定破解。"
]
```

---

## 新增球员 Checklist

- [ ] `profile_text_cn`：3-4 句，不含模板对比，不含情感词汇
- [ ] `ceilingTemplate`：上限 NBA 对比 + 一句兑现条件
- [ ] `floorTemplate`：下限 NBA 对比 + 一句下滑风险
- [ ] `strengths_cn`：3-4 条，有具体数据或百分位
- [ ] `weaknesses_cn`：3-4 条，具体可验证，非空泛套话
- [ ] 已有的 `attributes` / `combine` / `tags` 字段完整

## UI 渲染对应关系

| 字段 | 页面位置 |
|---|---|
| `profile_text_cn` | 球员详情 → 概览 → 球员综述 |
| `ceilingTemplate` | 球员详情 → 概览 → 上限模板卡片 |
| `floorTemplate` | 球员详情 → 概览 → 下限模板卡片 |
| `strengths_cn` | 球员详情 → 概览 → 优势 |
| `weaknesses_cn` | 球员详情 → 概览 → 待观察 |

所有字段通过 `adaptDraftPlayer()` 在 [PlayerProfile.tsx](../../src/app/components/PlayerProfile.tsx) 中读取，不存在则走 fallback：
- strengths → `tags` 前 3 个翻译
- weaknesses → 通用占位文本
- ceiling/floor → 「AI 分析中」
