# Next Star — A/B 实验设计

## 实验：位置过滤 vs 位置加权

### 背景

V1.0 的匹配逻辑：只推荐用户所选位置的球员（选 PG → 只看 PG）。

问题：如果本届最好的发动机恰好是 SG，PG 用户永远看不到。位置相同不等于风格相同——一个防守型中锋和一个组织型中锋差异可能大于跨位置球员。

### 假设

> 位置加权（同位置 +15% 加分但不排除其他位置）比位置过滤（仅同位置）带来更高的推荐点击率和关注转化率。

### 实验设计

| 维度 | 说明 |
|------|------|
| 对照组 (A) | 位置过滤：只推荐同位置 Top 4 |
| 实验组 (B) | 位置加权：全部球员排名，同位置 +15% 加成，多样性约束 |
| 分流方式 | 用户 ID 哈希取模，50/50 |
| 样本量 | 每组至少 100 名完成入职的用户 |
| 主要指标 | 推荐点击率（点击至少 1 个推荐 / 看到推荐页） |
| 次要指标 | 关注转化率、推荐结果位置多样性（跨位置推荐数） |
| 护栏指标 | 入职完成率（不应因推荐逻辑变化而下降） |

### 预期结果

- B 组推荐点击率 > A 组（因为推荐池更大，可能包含意外但合理的匹配）
- B 组位置多样性 > A 组（跨位置推荐数 >0）
- A 组同位置满意度可能略高（但总点击率低）

### 成功标准

如果 B 组推荐点击率显著高于 A 组（p<0.05），则全量切换为位置加权。

如果 B 组不显著优于 A 组，考虑折中方案：Top 4 中至少 2 个同位置 + 2 个开放。
